Wiki Release Troubleshooting

Linux

Linux, X11, and Wayland.

Aero P2P Chat is shipped as a native Executable on Linux. The app uses Tauri with WebKitGTK, so the installed system WebKitGTK stack matters more than it does on Windows.

Install

Recommended installer command:

bash <(curl -sSL https://zorblock.github.io/AeroP2Pchat/install.sh)

After install, these commands are available when `~/.local/bin` is in PATH:

aerop2p open aerop2p status aerop2p update aerop2p uninstall

Check X11 or Wayland

Run this before starting the app:

echo $XDG_SESSION_TYPE

`x11` and `wayland` are both expected to work. If one session has compositor or driver issues, test the other session from your login screen.

Black or empty window

A black window usually means WebKitGTK could not render the frontend, or the Linux Executable could not resolve bundled frontend assets. Current releases build the frontend with relative assets and a conservative WebKit-compatible target.

Useful test command:

WEBKIT_DISABLE_COMPOSITING_MODE=1 ./Aero-P2P-Chat-Linux.x86_64

If that works, the issue is usually GPU/compositor related. Keep the environment variable in your launcher while debugging the distro graphics stack.

Direct Linux Executable start

chmod +x Aero-P2P-Chat-Linux.x86_64 ./Aero-P2P-Chat-Linux.x86_64
