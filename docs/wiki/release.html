Release

Release and update flow.

Releases publish a Windows setup installer and a Linux AppImage. The app and installer scripts use `latest.yml` to discover current assets and checksums.

Version rules

The project follows the existing `26.x.x` version system. A normal release bumps the minor version, while patch releases bump only the patch number.

npm run release npm run release -- --patch npm run patch

Generated release assets

  • Windows: `Aero-P2P-Chat-Windows-Setup.exe`
  • Linux: `Aero-P2P-Chat-Linux.AppImage`
  • Update metadata: `latest.yml`
  • Platform manifests: `update_manifest_windows.json` and `update_manifest_linux.json`

Workflow behavior

The Linux release workflow is only started by the release script via `workflow_dispatch`. It is not configured to run on every push.

The workflow builds only the Linux AppImage. Windows is built locally before the Linux workflow starts.

Updater behavior

Windows can download and start the setup installer from inside the app. Linux shows the installer command so users can run the AppImage update through the shell helper.
